## 第二阶段：支付验证 + H5 账单（第 7–10 周）

### 第 7–8 周：Slip 上传与支付验证

**Cloudflare Worker 代理层（租客接口）**
- [ ] `GET /b/:share_token` — Worker 校验 token_hash + 过期，返回脱敏账单字段
- [ ] `POST /b/:share_token/slip` — Slip 上传接口（频率限制 + 文件大小限制 + MIME 白名单 + EXIF 清理）
- [ ] 上传 Slip 到 R2，存储 object_key

**Slip2go 集成**
- [ ] 对接 Slip2go API（verify 接口）
- [ ] 支付状态机流转：`uploaded → verifying → verified / rejected`
- [ ] 降级为人工确认流程：`uploaded → manual_approved / manual_rejected`
- [ ] 账单状态推导逻辑（`verified/manual_approved` → `paid`；超期未付 → `overdue`）

**房东端支付管理**
- [ ] 账单列表展示支付状态
- [ ] 人工确认操作（approve / reject）
- [ ] Slip 图片查看（R2 预签名 URL）

## 关键依赖与风险备忘

| 依赖项 | 风险 | 应对 |
|--------|------|------|
| Slip2go API | 服务中断 / 定价变动 | 降级为人工确认，预留成本控制开关 |
| Thai-address 库 | 数据不全 / 停止维护 | 备选 Google Places API（需纳入成本） |
| Cloudflare Workers Cron | 提醒调度能力有限 | 评估是否需要 Upstash QStash 等外部调度 |
| Slip2go 成本 | 按调用计费，规模放大后敏感 | 预留"仅高风险触发核验"开关 |
