核心业务逻辑
1. 资产与楼盘建模

    地址管理： 集成 Thai-address 库，支持 Moo, Soi, Tambon 等泰国特色地址自动补全。

    灵活租赁模式： 支持月租（长租）、日租（短租）、钟点房、以及夜市摊位（计费）。

    脱敏管理： 数据库仅存储房间标签（Room Label），保护物理隐私。

2. 水电费动态计算逻辑 (三元组互推)

系统支持在以下三个变量中，只要输入任意两个，自动计算第三个：

    用量 (Usage): 本次读数−上次读数

    单价 (Unit Price): 系统预设（如 8 泰铢/度）。

    金额 (Amount): 用量×单价

    下次生成账单时，系统自动调取上次的 Current Reading 作为本次的 Previous Reading。

3. 自动化支付流水线

    生成链接： 房东生成 H5 账单链接发送至租客（Line/WhatsApp/SMS）。

    动态渲染： 租客端 H5 根据浏览器语言自动切换中/英/泰。

    Slip 核实： 租客上传转账单（Slip），后端通过 Slip2go API 自动核实金额、时间，验证通过后自动修改房间状态。

技术架构 (The Tech Stack)

EZBill 采用极致的 Serverless（无服务器） 架构，确保低成本、高性能与全球加速。
模块	技术选型	核心作用
前端 (房东)	Cloudflare Pages	部署 TypeScript + React/Vue，GitHub 一键部署，支持手机自适应。H5（Typescript）
后端 API	Cloudflare Workers	处理业务逻辑、调用 Slip 验证 API。
数据库	Supabase Pro	PostgreSQL 存储，Schema 隔离（救援 APP 与 EZBill 独立）。
文件存储	Cloudflare R2	存放租客上传的 Slip 原图（0 出行流量费）。
